目的
- 管理画面に「ホーム」ページを追加し、空のプレースホルダページを表示することで、今後の管理機能拡張のためのルートとレイアウトを確保する。

範囲
- /admin または既存の管理ルート配下にホーム用ルートを追加する実装（ルーティング、コントローラ、ビュー/コンポーネント）。
- 既存の認証・レイアウトには影響を与えないこと。

要件
1. 指定された管理ルート（例: /admin または /admin/home）にアクセスすると、空のホームページが表示されること。
2. ページは少なくとも識別用のタイトル（例: 「管理ホーム」）を表示すること。
3. 実装は Rails のコントローラ + ビュー、またはプロジェクトの構成に合わせて Inertia+React コンポーネントで行うこと。
4. レスポンスは HTTP 200 を返すこと。

受け入れ基準
- 管理ホームのルートへ GET リクエストを送信し、HTTP 200 が返ること。
- 表示されるページに「管理ホーム」などのタイトルが確認できること。
- 既存の画面や認証に悪影響がないこと。

技術的制約
- 最小限の変更で実装すること。
- 必要に応じて簡易的なリクエストスペックや統合テストを追加すること。
