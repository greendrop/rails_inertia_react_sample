# 要件: 管理画面の記事一覧にページネーションを追加する

作成日時: 2025-12-11T07:14:07.512Z

目的
- 管理画面の「記事一覧」にページネーションを導入し、大量の記事でも表示や操作が容易になるようにする。

対象画面
- 管理画面: 記事一覧（管理者が記事を閲覧・編集・削除できる一覧画面）

機能要件
1. サーバーサイドページネーション
   - ActiveRecordのlimit/offsetを用いるか、Kaminari/will_paginateを導入してページネーションを実装する。
2. ページングAPI情報
   - レスポンスに以下を含める: current_page, total_pages, total_count, per_page。
3. フロントエンド表示
   - Inertia + React の管理画面で以下のUIを実装する: 前へ/次へボタン、ページ番号、現在ページの強調。
   - 現在のページ番号はクエリパラメータで保持し、URLで再現可能とする。
4. デフォルト件数
   - 1ページあたりのデフォルト表示件数は20件とする。管理者が変更可能な設定はオプション扱い。
5. テスト
   - コントローラ（またはAPI）とシステムテストで、ページング動作（ページ遷移、境界条件、合計件数反映）を検証する。

非機能要件
- N+1問題回避のため必要な関連は事前にincludesで読み込むこと。
- アクセシビリティ（キーボード操作、スクリーンリーダーの順序）に配慮する。
- 性能面では大規模データでも応答時間が許容できるようにする。

備考
- 既存の管理画面デザインに合わせる。実装ライブラリは設計フェーズで決定する。
