# 要件: 管理画面の記事一覧にページネーションを追加する

作成日時: 2025-12-11T07:05:19.729Z

目的:
- 管理画面の「記事一覧」画面にページネーションを追加し、大量の投稿でも表示を分割して読み込み/操作しやすくする。

要件:
- サーバーサイドページネーションを実装すること（ActiveRecordのlimit/offsetまたはKaminariの利用）。
- 管理画面の既存UIにページネネーションUIを追加する（前へ/次へ、ページ番号）。
- デフォルトの1ページあたり表示数は20件。管理者が変更できる設定はオプションとして検討する。
- Inertia/React側で現在のページをクエリパラメータで保持し、ページ遷移時にURLで再現できること。
- API/コントローラはページ情報（current_page, total_pages, total_count, per_page）を返すこと。
- テスト: コントローラ/システムテストでページネーションの動作を検証する。

非機能要件:
- パフォーマンスを考慮して不要なN+1が発生しないようにする。
- アクセシビリティに配慮したキーボード操作対応。

その他:
- 既存コードやデザインに合わせる。実装案や利用ライブラリは設計フェーズで決定する。
