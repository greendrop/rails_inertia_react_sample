# ユーザー画面で記事一覧を表示する（user_site_articles_index）

## 概要
- ログイン済みユーザーが、公開されている記事の一覧をユーザー向けサイト上で閲覧できるようにする。

## 範囲
- 対象ページ: ユーザー向け記事一覧ページ（例: `/articles` またはそれに相当するパス）。
- 一覧表示のみを対象とし、詳細表示・作成・編集・削除は別 spec で扱う。
- ページネーションや検索・絞り込みは本要件には含めず、必要であれば別 spec で扱う。

## 要件
1. ユーザー画面の「記事一覧」ページにアクセスすると、公開状態の記事のみが一覧表示されること。
2. 各記事行には、タイトル、本文の冒頭または概要、公開日時（published_at）を表示すること。
3. 一覧は公開日時の降順（新しい順）で表示されること。
4. タイトルをクリックすると、対応する記事詳細ページに遷移できるリンクになっていること。
5. 記事が1件も存在しない場合、「表示できる記事がありません」等の空状態メッセージを表示すること。

## 非機能要件
- UI は Inertia + React で実装し、Rails コントローラから Inertia 経由で Article データを渡すこと。
- Article モデル（.kiro/specs/article_model）で定義されたフィールドおよび公開状態の定義に従うこと。
- 一覧取得および表示は、通常の件数でストレスなく閲覧できる応答時間であること。

## 受け入れ基準
1. 公開状態の記事が存在する場合に記事一覧ページへアクセスすると、公開記事のみが要件を満たす形式で表示されること。
2. 各記事タイトルをクリックすると、対応する記事詳細ページへ遷移すること。
3. 公開記事が存在しない状態で記事一覧ページへアクセスすると、空状態メッセージが表示されること。
4. 非公開記事や下書き記事が存在しても、ユーザー向け記事一覧には表示されないこと。

## 備考
- URL やナビゲーション上の配置など、詳細な UI デザインは別途調整する。
- ページネーション、検索、並び替えが必要になった場合は専用の spec（例: user_site_articles_pagination）で扱う。
