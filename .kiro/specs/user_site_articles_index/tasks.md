# タスク: ユーザー画面で記事一覧を表示する（user_site_articles_index）

作成日時: 2026-01-29T21:10:57.639Z

## 実装タスク一覧

- [ ] 1. ユーザー向け記事一覧ページ用のルーティングを追加する
  - ユーザーサイト向けのパス（例: `/articles`）に記事一覧ページを紐づける。
  - 既存のユーザーサイト用ネームスペースやルーティング方針に合わせて定義する。

- [ ] 2. 公開記事のみを取得して Inertia に渡すサーバーサイド処理を実装する
  - Article モデルから公開状態（published）の記事のみを公開日時降順で取得するクエリを用意する。
  - 取得した記事からフロントエンド表示用のデータ（タイトル、本文冒頭、公開日時）を整形し、Inertia 経由でフロントエンドに渡す。

- [ ] 3. ユーザー向け記事一覧ページの React コンポーネントを実装する
  - 受け取った記事一覧データを用いて、カードまたはリスト形式で記事を表示する。
  - 記事タイトルをクリックしたときに記事詳細ページへ遷移できるよう、リンクとして表示する。

- [ ] 4. 空状態および非公開記事の除外を確認する
  - 一覧対象となる記事が存在しない場合に、空状態メッセージ「表示できる記事がありません」が表示されることを確認する。
  - 下書き（draft）など公開状態でない記事がユーザー向け一覧に表示されないことを確認する。

- [ ] 5. ユーザー向け記事一覧ページのテストを追加する
  - 公開記事が存在する場合にタイトル・概要・公開日時が表示されることを確認するテストを追加する。
  - 公開記事が存在しない場合に空状態メッセージが表示されることを確認するテストを追加する。
  - 非公開記事が一覧に含まれないこと、およびタイトルから記事詳細ページに遷移できることを確認するテストを追加する。

## 備考
- ページネーションや検索、タグによる絞り込みは本タスクには含めず、必要に応じて別 spec で扱う。
